# 저장소 산출물 13분류

프로젝트 저장소 산출물을 아래 13범주로 분류한다. 파일 배치·정리·누락 점검 시 기준으로 사용.
경계가 애매하면 각 항목의 `↔` 구분 기준을 적용한다.

| # | 범주 | 목적 | 대표 항목 |
|---|---|---|---|
| 1 | 요구사항·기획 | 무엇을 어디까지 만드는가 | 요구사항 명세, 기능 정의서, 유스케이스, 마일스톤, 범위 정의 |
| 2 | 설계 자료 | 구조·규칙 사전 정의 | 아키텍처, ERD·DB **논리** 설계, API 명세, 시퀀스 다이어그램, 보안 설계 |
| 3 | 개발 소스 | 실제 코드 | 앱/FE/BE 소스, 공통 라이브러리, 배치, 연동 모듈 |
| 4 | 테스트 자료 | 테스트 수행 전체 | 단위/통합/E2E/성능/보안 테스트, 테스트 케이스, **fixture(실행 종속 입력값)**, 결과서 |
| 5 | 설정 파일 | 환경별 동작 제어 | 환경 설정, 앱 설정, DB 연결, **로깅 설정**, 기능 플래그, `.env.example` |
| 6 | 빌드·의존성 | 설치·실행 형태로 | `package.json`/`pom.xml`/`requirements.txt`, 빌드 스크립트, **의존성 라이선스 메타데이터** |
| 7 | 배포·인프라 | 서버·클라우드 배포 | `Dockerfile`, k8s, Terraform, `nginx.conf`, **마이그레이션 실행**, 배포/롤백 스크립트 |
| 8 | CI/CD | 자동 검사·빌드·배포 | `.github/workflows`, `.gitlab-ci.yml`, `Jenkinsfile`, 정적분석, 릴리스 자동화 |
| 9 | 운영 자료 | 오픈 후 관리 | 운영 매뉴얼, 장애 대응, 모니터링, **로그 수집**, 백업·복구, 점검 체크리스트 |
| 10 | 데이터·리소스 | 사용 데이터·정적 파일 | 초기/샘플 데이터, DB **스키마**, **마이그레이션 파일 정의**, 이미지·폰트, i18n, **Mock 정적 데이터** |
| 11 | 개발·운영 도구 | 반복작업 자동화 보조 | 코드 생성, **DB 초기화·시딩**, 데이터 변환, **린트·포맷 설정 파일**, Mock 서버, API 컬렉션 |
| 12 | 프로젝트 관리 | 협업·의사결정 | 일정, 회의록, 의사결정 기록, 이슈, 릴리스 계획, **저장소 문서(README·CHANGELOG·CONTRIBUTING)** |
| 13 | 보안·품질 기준 | 지켜야 할 규칙 | **코딩 컨벤션(규칙 자체)**, 리뷰 기준, 브랜치 전략, 개인정보 처리, **OSS 라이선스 점검 기준**, DoD |

## 경계 기준 (헷갈릴 때)

- **테스트데이터**: 실행 종속 fixture·입력값 = **4** / 앱이 참조하는 재사용·시드 데이터 = **10**
- **마이그레이션**: 스키마 변경 정의 파일 = **10** / 배포 실행 = **7** / 초기화·시딩 스크립트 = **11**
- **DB 설계**: 논리 설계(ERD) = **2** / 실행 스키마 파일(schema.sql·DDL) = **10**
- **설정 vs 인프라**: 앱 런타임이 읽는 설정 = **5** / 인프라 프로비저닝(`nginx.conf` 등) = **7**
- **라이선스**: 의존성 라이선스 메타데이터 = **6** / 컴플라이언스 점검 기준 = **13**
- **린트**: 규칙 집행 설정 파일(`.eslintrc`) = **11** / 규칙·기준 자체(컨벤션) = **13**
- **저장소 메타파일**(`.gitignore`, `.gitattributes`, `.editorconfig`, `LICENSE`, `CODEOWNERS`, `.github/` 템플릿): **5**(설정 파일)에 둔다.

## 관리 원칙

> 비밀값(비밀번호·API 키·인증서)은 저장소에 직접 넣지 않고 Secret Manager·CI 보안 변수로 관리.
> 대용량 원본 데이터는 코드 저장소 대신 별도 스토리지.
> Jira·Notion 등 외부 도구 관리 시 저장소엔 링크와 핵심 결정만.

이 스킬 본문이 분류 기준의 단일 출처(single source of truth)다.
